## 2023-10-27 - Cross-Site Scripting (XSS) in innerHTML Rendering
**Vulnerability:** Unsanitized user inputs (Leaderboard names/avatars) and API responses (Game Level Clues) were directly injected into the DOM using `innerHTML` in `ayocross.html`.
**Learning:** Vanilla JavaScript applications that rely heavily on string concatenation and `innerHTML` for rendering are highly susceptible to XSS.
**Prevention:** Always use a helper function like `escapeHTML` to sanitize any dynamic data before using `innerHTML`, or prefer using `textContent` for text-only updates.
\n## 2025-06-13 - Missing XSS Sanitization in SPA Views\n**Vulnerability:** User inputs (avatars, themes, duel names, profile history) were injected directly into `innerHTML` without sanitization.\n**Learning:** In a vanilla JS SPA, dynamically generated views (like Leaderboard, Profile, Duel Level) that use string templates and `innerHTML` need comprehensive XSS checking, not just on usernames but on all user-controlled data fields.\n**Prevention:** Ensure all dynamic data fields inside `innerHTML` template strings are wrapped with `escapeHTML()`, especially when adding new features or views.
