## 2024-05-24 - Canvas Animation Performance Optimization
**Learning:** The 'ocean' theme animation in `ayocross.html` recalculates and draws the sine wave for every single pixel horizontally (`x=0; x<canvas.width; x++`). For a standard 1080p display, this means calling `ctx.lineTo` ~1920 times per wave layer, times 3 layers = ~5760 calls per frame. This is extremely inefficient and can cause significant frame drops on lower-end devices. Similarly, particle systems (snow, sparkle, city) change `ctx.fillStyle` inside the particle loop for every particle, which forces context state changes for each particle, rather than batching them.

**Action:** Step sizes for wave drawing can safely be increased (e.g. `x+=20`) since sine waves are smooth, dramatically reducing the number of `lineTo` operations by 95% with no visual degradation. Context state changes (like setting `ctx.fillStyle` or shadow settings) should be hoisted outside the particle loops where possible to minimize state switching overhead.
